# 剧本Schema设计文档

## 一、概述

本文件定义项目统一短剧剧本 YAML 数据结构（Schema 数据契约）。所有大模型生成剧本必须严格遵循该固定格式，保证结构统一、后端可稳定解析、前端可正常渲染、文件可标准化导出。

## 二、基准 Schema 规范（AI 强制输出模板）

所有模型生成剧本，必须严格遵守以下固定字段与层级结构：

```yaml
# 剧本全局基础信息
title: 短剧标题                # 剧本整体名称
summary: 剧情简介（控制在30字左右） # 单段剧情核心概括

# 剧情场景列表（按剧情时间线排序）
scenes:
  - scene_id: S01              # 场景唯一编号，格式 S+两位数字
    location: 场景地点          # 剧情发生的具体场景位置
    time: 日/夜                # 场景时间基础属性，可细化时段（日/夜/黄昏/深夜）

    # 单场景分镜镜头列表（按播放顺序排列）
    shots:
      - shot_id: 1             # 镜头唯一序号，同场景内从1递增
        frame: 远景/全景/中景/近景/特写 # 标准影视景别
        duration: 数字          # 镜头时长，单位：秒
        visual: 画面详细描述    # 纯视觉画面、人物、动作、场景描写，不含声音
        audio: 角色名(情绪):台词 或 【环境音效】 # 台词/音效，纯声音内容，与画面严格分离
```

## 三、标准完整示例（带精准注释）

以下为可直接复用的标准成品剧本示例，完全贴合上述基准 Schema：

```yaml
# 剧本全局基础信息
title: 魔主方源：春秋蝉之围       # 短剧剧本整体标题，用于页面展示、文件命名
summary: 正道群雄围攻方源索要春秋蝉，方源浴血对峙夕阳下  # 30字左右剧情简介，概括核心冲突

# 剧情场景列表：按剧情时间线排序，单剧本可包含多个场景
scenes:
- scene_id: S01                   # 场景唯一ID，全局统一递增规则（S01、S02...）
  location: 山间悬崖空地           # 明确剧情发生的物理场景地点
  time: 日（黄昏）                 # 基础时间属性+细化时段，适配影视光影拍摄逻辑

  # 分镜镜头列表：单场景内最小播放单元，严格按播放顺序排序
  shots:
  - shot_id: 1                    # 同场景镜头序号，从1开始依次递增
    frame: 全景                   # 影视标准景别，规范镜头呈现效果
    duration: 5                   # 镜头播放时长，单位秒，控制成片节奏
    visual: 山间空地，正道群雄持兵器围成圈，中心方源浴血而立，破袍随风飘动  # 纯视觉画面描写，无任何声音内容
    audio: 【嘈杂人声】“交出春秋蝉！”“踏破魔窟！”“罪无可恕！”  # 群体人声音效+台词

  - shot_id: 2
    frame: 近景
    duration: 3
    visual: 方源披头散发，碧绿大袍残破染血，面无表情，目光幽深如古井
    audio: 【山风声】  # 纯环境氛围音效

  - shot_id: 3
    frame: 特写
    duration: 2
    visual: 方源脚下积起的暗红血水，映着夕阳微光
    audio: 【血滴声】  # 细节环境音效，强化画面沉浸感

  - shot_id: 4
    frame: 中景
    duration: 4
    visual: 几位正道人物神态各异：咆哮、冷笑、警惕眯眼、捂伤口发抖
    audio: 正道长老(愤怒):“方老魔，你别妄图反抗！” 女修(怨毒):“我要让你生不如死！”  # 带情绪标注的角色台词

  - shot_id: 5
    frame: 全景
    duration: 6
    visual: 夕阳西下，晚霞如火，群雄与方源对峙，影子被余晖拉长
    audio: 【山风声】【归鸟啼鸣】  # 多重环境音效叠加烘托氛围
```

## 四、字段详细释义

### 1. 全局根字段

- **title**：必填，剧本专属标题，用于页面展示、剧本文件下载命名
- **summary**：必填，30字左右精简剧情简介，概括单段剧情核心冲突
- **scenes**：必填，场景数组，存储全部剧情场景，严格按剧情时间线排序

### 2. 场景字段（scene）

- **scene_id**：场景唯一标识，固定格式 S+两位数字，全局唯一、依次递增
- **location**：剧情发生的具体场景地点，贴合小说原文设定
- **time**：场景时间属性，基础取值为日/夜，可细化黄昏、深夜等时段
- **shots**：镜头数组，存储单场景下所有分镜镜头，按播放顺序排列

### 3. 镜头字段（shot）

- **shot_id**：镜头唯一序号，同一场景内从1开始递增，区分不同分镜
- **frame**：镜头景别，仅支持：远景、全景、中景、近景、特写
- **duration**：镜头播放时长，纯数字，单位为秒
- **visual**：纯视觉内容，仅描写画面、人物、动作、场景，禁止包含声音、台词
- **audio**：纯声音内容，仅描写台词、环境音效，禁止包含画面描写；支持【音效】、角色(情绪):台词两种格式

## 五、强制输出约束规则

- 所有模型生成剧本，必须严格遵循本文档字段、层级、顺序，禁止缺字段、乱序、新增无效字段
- 严格执行**音画分离**：visual 只写画面，audio 只写声音，互不嵌套、互不重叠
- 所有 ID 严格有序递增，保证剧本结构化、可解析、可编辑、可复用
- summary 严格控制在30字左右，保持格式统一规范
